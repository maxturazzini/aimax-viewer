Artifacts Folder Not Found

AIMax Viewer looks for a Artifacts/ folder in your workspace to display HTML artifacts. This folder was not found in the current workspace or any parent directory.

Quick Setup

Follow these steps to set up AIMax Viewer in your workspace:

1

Create the folder structure

In your workspace root, create the following folders:

mkdir -p Artifacts
2

Create an index.html file

Create a home page for your artifacts dashboard:

Artifacts/index.html
3

Reload VS Code window

Press Cmd+Shift+P and run "Developer: Reload Window" to restart the extension.

Example index.html

Here's a minimal template you can use for your artifacts home page:

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <title>My Artifacts</title>
    <style>
        body {
            font-family: system-ui, sans-serif;
            background: #1e1e1e;
            color: #e0e0e0;
            padding: 40px;
        }
        h1 { color: #00d4ff; }
        a { color: #00d4ff; }
    </style>
</head>
<body>
    <h1>My Artifacts</h1>
    <p>Your HTML artifacts will appear in the dropdown above.</p>
</body>
</html>

How It Works

  • Artifacts folder: Place your HTML files in Artifacts/
  • Auto-discovery: All HTML files are listed in the toolbar dropdown
  • Sorted by date: Newest files appear first
  • Home page: index.html is your dashboard landing page

AI Agent Integration

When AI agents (like Claude Code) create HTML artifacts, they can open them directly in the viewer:

open "vscode://aimax.aimax-viewer/openBrowser?http://127.0.0.1:3124/Artifacts/my-artifact.html"

This opens the artifact inside VS Code with the full toolbar (reload, navigation, dropdown selector).
